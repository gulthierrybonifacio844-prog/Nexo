Sistema de Controle de Acesso Inteligente

Projeto Integrador — 2° Ano A — IFB Campus Ceilândia. Sistema criado para gerenciar a entrada de estudantes no campus usando hardware e software de baixo custo, com foco em IoT, eficiência e segurança.

Integrantes

Filipe Martins Fontinele, Gulthierry Bonifácio de Jesus, Jailson dos Santos Campelo, Mariana Gonçalves de Macedo Freitas, Pedro Gabriel Alves dos Santos.

O Desafio

Existe a necessidade de gerenciar a entrada de estudantes no Campus Ceilândia de forma segura e eficiente.

A Proposta

Criar um sistema inteligente de controle de acesso, integrando hardware e software, centralizado e escalável.

Diferencial

Projeto de baixo custo baseado em tecnologias abertas, com interface moderna e intuitiva.

Hardware Principal

Raspberry Pi 3 Model B — o cérebro do sistema.

  • Processador Quad‑Core 1.2 GHz, 1GB RAM.
  • Wi-Fi + Bluetooth integrados.
  • Baixo consumo e alta confiabilidade.
  • Hospeda o banco de dados, servidor e dashboard.

Software — Sistema Operacional

Raspberry Pi OS (baseado em Debian). Estável, leve, ideal para aplicações embarcadas e quiosques digitais.

Comunicação — Protocolo MQTT

Utilizado pela sua rapidez, baixa latência e funcionamento publish/subscribe.

Dashboard de Monitoramento

Interface acessada via navegador (PC ou celular). Exibe entradas em tempo real, cadastro de usuários, permissões e relatórios.

Interface na Prática

Tela de Login

Validação Técnica

Modelo validado por empresas como a B2Mídia, que utiliza o Raspberry Pi 3 em murais digitais.

Arquitetura e Capacidade

Hardware suporta servidor + interface sem travamentos. Arquitetura com Raspberry Pi + ESP32/Arduino torna o sistema escalável.

Conectividade

Wi-Fi integrado elimina cabos e permite comunicação eficiente entre servidor, sensores e leitores.

Próximos Passos — Curto Prazo

  • Integração física entre leitor ESP32 e Raspberry.
  • Testes de comunicação via MQTT.

Próximos Passos — Médio Prazo

  • Construção da caixa de proteção.
  • Testes reais com cartões de estudantes.

Conclusão

O sistema é viável técnica e financeiramente. Base totalmente funcional: servidor, dashboard e comunicação inicial.
